## 📌 Project Overview

Modern streaming platforms like Netflix and Spotify serve millions of users who face the "paradox of choice" — too much content, too little time. Recommendation systems solve this by surfacing relevant content automatically.

This project builds a **complete, end-to-end recommendation system** from scratch:

- Raw data → Data Cleaning → EDA → Feature Engineering → Model Building → Evaluation → Deployed Web App
- Three distinct recommendation approaches, each solving a different business problem
- Interactive Streamlit application where anyone can explore recommendations in real time

## 🚀 Features

### 🏆 Popularity-Based Recommendation
- Ranks movies by average rating with a minimum ratings threshold
- Eliminates small-sample bias (a movie with 2 perfect ratings ≠ a genuinely great movie)
- Best for: new users with no history

### 🎭 Content-Based Filtering
- Combines one-hot encoded genres (19 categories) with TF-IDF tag vectors (500 features)
- Uses cosine similarity to find movies similar in content and feel
- Best for: "I liked X, show me more like X"
- Handles the cold-start problem for new movies

### 👤 Collaborative Filtering (SVD)
- Matrix factorization via TruncatedSVD on a sparse 138,493 × 26,744 user-item matrix
- Learns 50 latent factors representing hidden taste dimensions
- Mean-centered ratings remove individual rating-scale bias
- Best for: users with an established rating history

## ⚙️ Installation

### Prerequisites
- Python 3.10+
- Git

### Steps

**1. Clone the repository**
```bash
git clone https://github.com/chamathkaridmi/movie-recommender.git
cd movie-recommender
```

**2. Create and activate a virtual environment**
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate
```

**3. Install dependencies**
```bash
pip install -r requirements.txt
```

**4. Download the dataset**

Download the MovieLens 20M dataset from [Kaggle](https://www.kaggle.com/datasets/grouplens/movielens-20m-dataset) and place the CSV files in `data/raw/`:
```
data/raw/
├── rating.csv
├── movie.csv
├── tag.csv
└── link.csv
```

**5. Run the notebooks in order**

Open Jupyter and run notebooks `01` through `05` in sequence. Each notebook saves its outputs to `data/processed/` and `data/features/`, which the app loads at startup.

**6. Launch the application**
```bash
streamlit run app/app.py
```

The app will be available at `http://localhost:8501`

## 🎮 Usage

### Popularity-Based
1. Select **"Popular Movies"** in the sidebar
2. Adjust the minimum ratings threshold

### Content-Based

2. Enter any User ID between 1 and 138,493
3. Click **"Get My Recommendations"**

## 📈 Model Evaluation Results

### Rating Prediction (SVD)

| Metric | SVD (Centered) | Baseline (Mean) | Improvement |
|---|---|---|---|
| RMSE | **0.9057** | 1.0522 | 13.9% ↑ |
| MAE | **0.6969** | 0.8413 | 17.2% ↑ |

### Ranking Quality (Precision@K & Recall@K)

*Evaluated on 500 sampled users, relevance threshold: rating ≥ 4.0*

| Model | P@5 | P@10 | R@5 | R@10 |
|---|---|---|---|---|
| Popularity-Based | 0.0516 | 0.0350 | 0.0244 | 0.0327 |
| Content-Based | 0.0073 | 0.0077 | 0.0023 | 0.0045 |
| **SVD (Collaborative)** | **0.1858** | **0.1419** | **0.0810** | **0.1155** |

### Production Recommendation

A **hybrid cascade strategy** is recommended for real-world deployment:

| User Situation | Recommended Model |
|---|---|
| New user (no history) | Popularity-Based |
| Light user (1–5 ratings) | Content-Based |
| Active user (20+ ratings) | Collaborative Filtering |
| New movie (zero ratings) | Content-Based |

## 📊 Key Data Science Findings

From the Exploratory Data Analysis phase:

- **Rating distribution** is left-skewed (mean 3.53, mode 4.0) — users are generous raters
- **Power-law concentration**: top 0.073% of movies absorb 5.26% of all ratings
- **Popularity ≠ Quality**: only 1 movie (Shawshank Redemption) appears in both top-rated AND most-rated lists
- **99.46% sparsity** in the user-item matrix — the central challenge for collaborative filtering
- **Film-Noir**: least popular genre by volume yet highest average rating — self-selecting enthusiast audience
